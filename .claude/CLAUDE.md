# Carbon - PHP DateTime Extension

## Назначение проекта

Carbon - это расширение для PHP DateTime API, предоставляющее удобный и выразительный интерфейс для работы с датами и временем. Библиотека позволяет легко выполнять операции с датами, форматировать их, сравнивать, вычислять разницу и многое другое на более чем 200 языках.

**Основные сценарии использования:**
- Создание и парсинг дат с удобным API
- Манипуляции с датами (добавление/вычитание дней, месяцев, лет)
- Форматирование дат в различных форматах (ISO, RFC, человекочитаемый)
- Сравнение дат и вычисление разниц
- Работа с временными зонами
- Локализация (перевод дат на разные языки)
- Тестирование с мокированием текущего времени

**Как запустить:**
```bash
# Установка зависимостей
composer install

# Запуск тестов
./vendor/bin/phpunit

# Проверка стиля кода
./vendor/bin/php-cs-fixer fix --dry-run --diff

# Статический анализ
./vendor/bin/phpstan analyse
```

### Ответственность модулей

**src/Carbon/Carbon.php:**
- Главный класс, расширяет DateTime
- Реализует все основные методы работы с датами
- Поддерживает fluent interface

**src/Carbon/CarbonImmutable.php:**
- Иммутабельная версия Carbon
- Каждая операция возвращает новый объект
- Для функционального программирования

**src/Carbon/Traits/:**
- **Creator.php** - создание дат из различных форматов
- **Difference.php** - вычисление разниц между датами
- **Comparison.php** - методы сравнения дат
- **Modifiers.php** - модификация дат (add, sub, set)
- **Converter.php** - конвертация в различные форматы
- **Test.php** - утилиты для тестирования (mock)
- **Date.php**, **Week.php**, **Month.php**, etc. - специфичные операции

**src/Carbon/Lang/:**
- Файлы локализации для 200+ языков
- Каждый язык в отдельном файле (например, ru.php, en.php)
- Содержат переводы для месяцев, дней недели, относительных дат

**tests/:**
- Полное покрытие тестами (~95%+)
- Тесты организованы по функциональности
- Используют PHPUnit и data providers

## Ключевые термины (мини-глоссарий)

- **Carbon** - основной класс для работы с датами (mutable)
- **CarbonImmutable** - иммутабельная версия Carbon
- **CarbonInterval** - представление временного интервала (например, "2 дня 3 часа")
- **CarbonPeriod** - представление периода с итерацией (например, "с 1 января по 31 марта с шагом 1 день")
- **Mutable/Immutable** - изменяемый/неизменяемый объект
- **Fluent Interface** - цепочка методов ($date->addDay()->subHour())
- **Locale** - локаль для локализации (ru, en, fr, etc.)
- **Timezone** - временная зона (Europe/Moscow, UTC, etc.)
- **Macro** - пользовательский метод, добавленный в Carbon через макросы
- **TestNow** - мокированное "текущее время" для тестирования

## Требования к результату

### Стиль кода и форматирование

**Обязательные требования:**

1. **PSR-12 Code Style**
   - Использовать PHP-CS-Fixer с конфигурацией `.php-cs-fixer.dist.php`
   - 4 пробела для отступов
   - Unix line endings (LF)
   
2. **Type Hints**
   - Все параметры методов должны иметь type hints
   - Все методы должны иметь return type hints
   - Использовать union types где уместно (PHP 8.0+)
   
3. **PHPDoc комментарии**
   - Все публичные методы должны иметь PHPDoc
   - Описание должно быть кратким и понятным
   - Указывать @throws для исключений
   - Указывать @deprecated для устаревших методов
   
4. **Именование**
   - camelCase для методов и переменных
   - PascalCase для классов
   - UPPER_CASE для констант
   - Говорящие имена (не `$a`, а `$date` или `$timestamp`)

**Пример правильного кода:**


### Требования к тестам

**Все новые функции ДОЛЖНЫ иметь тесты:**

1. **Покрытие:**
   - Минимум 90% покрытие кода тестами
   - Покрытие всех граничных случаев

   - Один test case класс на один класс источника

3. **Именование тестов:**
   - Префикс `test` обязателен
   - Имя должно описывать что тестируется
   - Пример: `testAddDaysReturnsCorrectDate()`
   
4. **Data Providers:**
   - Использовать для параметризованных тестов
   - Каждый набор данных должен иметь описательный ключ

**Пример правильного теста:**

```php
<?php

declare(strict_types=1);

namespace Tests\Carbon;

use Carbon\Carbon;
use PHPUnit\Framework\TestCase;

class CreateTest extends TestCase
{
    /**
     * @dataProvider provideCreateData
     */
    public function testCreateWithValidParameters(
        int $year,
        int $month,
        int $day,
        string $expectedFormat
    ): void {
        $date = Carbon::create($year, $month, $day);
        
        $this->assertInstanceOf(Carbon::class, $date);
        $this->assertSame($expectedFormat, $date->format('Y-m-d'));
    }
    
    public static function provideCreateData(): array
    {
        return [
            'simple date' => [2024, 1, 15, '2024-01-15'],
            'leap year' => [2024, 2, 29, '2024-02-29'],
            'end of year' => [2024, 12, 31, '2024-12-31'],
        ];
    }
}
```
